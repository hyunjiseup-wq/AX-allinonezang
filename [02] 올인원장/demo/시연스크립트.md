# 시연 스크립트 — "CSV 넣으면 정제→생성까지 끊김 없이 이어진다"

> **업데이트**: 실제 발표에서는 아래 Claude Code 대화 방식 대신, `demo` 브랜치의
> **시연용 Streamlit**(`시연용-스트림릿/app.py`, 로컬 실행 권장)으로 정제→3채널 생성을 한 화면에서
> 보여준다. Claude API를 실시간 호출하는 방식은 현장 매니저 확인 하에 진행. 아래 원래 스크립트는
> Q&A에서 "판단 과정을 코드로도 볼 수 있나요?" 같은 질문이 나왔을 때의 백업 시연으로 남겨둔다.

발표 슬라이드 4번(산출물 & 실증) 또는 Q&A에서, "새 CSV를 넣어도 정말 재현되나요?"에 말로 답하는
대신 그 자리에서 직접 보여주기 위한 스크립트. 핵심은 **정제와 생성을 한 자리에서 끊김 없이 이어서
보여주는 것** — 스트림릿 화면과 채팅창을 오가는 게 아니라, Claude Code 대화 하나가 "CSV 업로드→
정제→3채널 생성"을 전부 처리하는 흐름을 그대로 재현한다.

## 준비물

- `demo/live_demo_input.csv` — 처음 보여주는 메인 데모용 CSV (5건, 완전중복 1건·미등록 톤 1건·
  결측 1건 포함)
- `demo/dummy_keywords_input.csv` — Q&A에서 "다른 CSV로도 되나요?"라고 물으면 바로 이어서 보여줄
  두 번째 CSV (8건, 더 다양한 엣지케이스 포함)
- 두 CSV 모두 사전 검증 완료, 정답은 `demo/generated_samples.md`에 있음

## 실행 순서 (한 자리에서 끊김 없이)

1. **(선택) 스트림릿으로 정제만 먼저 눈에 보여주기**
   `시연용-스트림릿` 앱에 `demo/live_demo_input.csv`를 업로드해 "5건 → 4건, 중복 1건 제거,
   결측 1건 보완" 결과가 화면에 뜨는 걸 보여준다. 이건 "정제가 결정론적 스크립트로 즉시 처리된다"는
   것만 보여주는 가벼운 프리뷰이고, 여기서 멈추지 않는다.

2. **바로 이어서 Claude Code 대화에서 정제 실행**
   ```
   python scripts/clean_keywords.py demo/live_demo_input.csv
   ```
   ```
   입력 5건 → 정제 후 4건
   중복 제거: 1건
   tone_hint 정규화: 0건
   detail 보완: 1건
   ```
   - id 204(id 201과 완전 중복) 제거됨
   - id 205 detail 결측 → 자동 보완됨
   - "설레는"(id 205 tone_hint)은 사전에 없는 값이라 tone_hint 정규화 0건으로 집계됨 —
     **억지로 매핑하지 않고 원본을 유지**한다는 걸 여기서 먼저 짚어준다.

3. **멈추지 않고 곧바로 3채널 생성 요청**
   정제된 4건(201·202·203·205)을 그대로 Claude Code 대화창에 붙여넣고 "이 케이스들로 3채널
   초안 만들어줘"라고 요청한다. 미리 준비한 답을 읽는 게 아니라 *그 자리에서*
   `output/channel-strategy.md` 규칙이 적용되며 생성되는 과정 자체가 핵심이다.
   - id 205("설레는")를 생성할 때, Claude가 매핑표에 없는 톤을 어떤 근거로 판단해 처리하는지
     말로 설명하게 하면 가장 설득력 있다 (이번 검증에서는 "밝은" 계열로 판단했다 — 1차 검증 때
     "차분한"을 "따뜻한" 계열로 판단한 것과는 다른 결과인데, 이 차이 자체가 "고정 규칙표를 그대로
     찾아 붙이는 게 아니라 매번 문맥에 맞게 판단한다"는 증거가 된다)
   - 결과가 채널별 길이 기준(블로그 500자+, 인스타 150자 이내+해시태그 5개, 카카오 180자
     이내+이모지+링크)을 충족하는지 그 자리에서 글자 수를 같이 세어 보여준다
   - 참고 답안(사전 검증 완료, 전문 그대로): `demo/generated_samples.md` "2차 라이브 검증" 섹션

4. **(Q&A 대비) 다른 CSV로도 되는지 물어보면**
   `demo/dummy_keywords_input.csv`(8건, "친근스러운"·"긴박"·"자랑스런"·"차분한" 등 더 다양한
   표기 흔들림 포함)로 2·3단계를 한 번 더 반복한다. 참고 답안: `demo/generated_samples.md` 1차 섹션.

## 왜 "끊김 없이 이어서" 보여주는 게 중요한가

- 정제(2단계)는 결정론적 스크립트라 항상 같은 결과가 나오고, 생성(3단계)은 Claude Code 대화가
  규칙을 적용해 판단하는 구간이다 — 이 둘이 "하나의 파이프라인"이라는 걸 보여주려면 중간에
  다른 화면으로 전환하지 않고 이어 붙여야 설득력이 있다.
- 웹 UI(스트림릿)는 정제 확인용 프리뷰로만 쓰고, 실제 "자동화"의 핵심인 생성은 항상 Claude Code
  대화 안에서 끝난다 — 이래야 "외부 API 호출 앱 금지"라는 과제 제약과 충돌하지 않으면서도,
  체감상으로는 CSV 하나로 전부 이어지는 자동화처럼 보여줄 수 있다.
- 미리 준비된 결과물을 읽는 것보다, **한 번도 보지 않은 데이터**로 그 자리에서 돌려서 같은 품질이
  나온다는 걸 증명하는 쪽이 신뢰도가 훨씬 높다.
- 실패 케이스(사전에 없는 톤)를 일부러 넣어서, 무너지지 않고 매번 합리적으로 처리되는 걸 보여주는
  것도 "엣지케이스까지 고려했다"는 신호가 된다.
